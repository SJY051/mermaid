# Claude Design 브리핑 — mermAid 모바일 셸

`claude.ai/design`에서 이 리포(public: `SJY051/mermaid`)를 코드베이스로 연결해 디자인을 생성할 때 함께 주는 브리핑입니다. **순수 시각·모션은 자유롭게 채우되, 안전 규칙은 협상 불가**입니다. Claude Design은 화면을 아름답게 만드는 데 강하고, 바로 그 과정에서 아래 §2 불변조건을 무심코 어기기 쉽습니다 — 그래서 이 문서의 절반은 "하지 말 것"입니다.

핸드오프 번들은 Claude Code로 넘겨 구현하며, 완료 기준은 [AGENTS.md](../../../AGENTS.md) §4(브라우저에서 실제로 클릭해 확인)입니다.

## 1분 컨텍스트

한국의 **영어권 사용자**가 로그인 없이 증상을 설명하면 → **정부 데이터로 검증된**(식약처·심평원·국립중앙의료원) 의약품 정보를 영어로, 그리고 **지금 영업 중인** 근처 약국·병원 지도를 받습니다. 핵심 단어는 **verified** — 앱은 AI가 아는 걸 옮기지 않고, 정부가 돌려준 사실만 설명합니다. 시각 언어는 **무채색 바탕 + 의미 컬러**입니다.

## 협상 불가 — AGENTS.md §2 (어기면 아픈 사람이 다칠 수 있음)

Claude Design이 "더 보기 좋게" 하려다 넘기 쉬운 순서로 적습니다.

- **`no_match_found`를 안심 신호로 그리지 말 것 (§2-2).** 알레르기 판정은 `blocked`/`warning`/`no_match_found`/`unknown` 네 상태입니다. `no_match_found`는 "안전"이 아니라 "우리가 가진 성분 목록에 없음"입니다. **초록 배지 금지, "safe"라는 단어 금지**, 알레르기 상태 근처에 초록색을 두지 말 것. 알레르기가 선언되면 모델이 제안한 약은 화면에서 아예 사라집니다(SA-08) — 대체약을 그리지 말 것.
- **`isOpenNow: null`은 "Closed"가 아니라 "Hours unknown" (§2-3).** null은 "판단 불가"입니다. 닫힘으로 그리면 밤에 아픈 사람을 열려 있는 약국 앞에서 돌려보냅니다.
- **면책과 응급은 항상 화면에 (§2-1).** "일반 정보이며 진단이 아님 · 응급? 119"는 모든 탭에 상시 노출. 응급(Call 119) 경로는 절대 가리거나 접지 말 것. 앱은 진단하지 않습니다.
- **색 = 의미, 장식 금지.** 빨강 = 응급·알레르기 차단 **뿐**, 초록 = "지금 영업 중" **뿐**, 호박 = 불확실. 그 외 전부 무채색. 브랜드감을 위해 컬러를 뿌리지 말 것 — 여기서 색은 안전 신호입니다.
- **상태 표시는 이모지가 아니라 아이콘 세트로.** `✓`/`?`/`✕`/`⛔` 같은 이모지는 폰트 폴백에 따라 깨지거나 다르게 렌더돼 **안전 정보 전달이 실패**할 수 있습니다. SVG/아이콘폰트로, 의미가 기기와 무관하게 고정되게.
- **접근성은 안전입니다.** 텍스트 대비 4.5:1(라이트·다크 **둘 다**, 전수), 상태를 **색만으로** 구분하지 말 것(글리프 + 접근 가능한 텍스트 병행), 한국어 고유명사는 `lang="ko"`(라벨에 들어가면 `aria-labelledby`로 분리 — `aria-label`은 `lang`을 못 실음).
- **출처(provenance)는 서버 소유 (§2-9).** 모든 fact 카드에 "어느 기관 · 언제". 픽스처 데이터를 실데이터처럼 그리지 말 것.

이 규칙들의 원문과 근거는 [AGENTS.md §2](../../../AGENTS.md#2-invariants), 렌더링 정본은 [`wireframe-v2.html`](wireframe-v2.html)입니다.

## 채워달라 — spec §7의 B절, 그리고 A·C의 빈 화면

[spec.md §7](spec.md)이 갭을 "누가 채우느냐"로 나눴습니다. Claude Design이 우리 규칙 위에서 채울 것:

**B. 시각·모션 (자유롭게)**
- 디자인 토큰: 타이포 스케일·간격 그리드·라운딩/그림자, astryx 토큰 매핑
- 아이콘/픽토그램 세트: 상태 글리프, 탭바 아이콘, 지도 핀 — 핀은 기하학적 ●■◆ 대신 **의미 픽토그램**(약국·병원·응급), 단 규칙(모양=종류·채움=상태·비색 글리프) 유지
- 모바일 크롬: safe-area(노치·홈), 44pt 터치 타겟, 키보드 레이아웃
- 마이크로인터랙션: 로딩·저장·전환·핀 선택

**A·C. 아직 안 그려진 화면 (안전 관점은 우리가 명세, 시각은 외주)**
- 약 카드 4-state(`warning`·`no_match_found`·`unknown` — 위 §2-2 준수)
- 에러·빈·무결과 화면(약 0건·시설 0개·위치 실패·API 다운)
- 다크 모드 전 화면(특히 응급 빨강)
- 내 위치 마커(+권한 거부 폴백)
- 최초 진입·온보딩, 탭 전환·딥링크·바텀시트 흐름
