1mermAid mobile shell — wireframe-v2 baseline, §7 gaps filled (B tokens/icons/chrome/motion · A safety screens · C flows)
Grounding
Visual baseline: wireframe-v2.html (all English UI copy verbatim from it, or from App.tsx / AllergyBadge.tsx). Data shapes & error codes from lib/types.ts. Korean appears only as proper nouns (drug · facility · agency names), wrapped lang="ko". Screens are 390×844, lang="en".
Non-negotiables applied everywhere
Red = act now (emergency · allergy block) only. Green = "open now" only — never near allergy states, never on toggles (§2-2). Amber = uncertainty. no_match_found stays plain neutral text — no badge, never "safe". isOpenNow:null renders "Hours unknown", never "Closed" (§2-3). Disclaimer strip + Call 119 path on every tab, AA ≥ 4.5:1 in both modes. Status glyphs are SVG, never emoji. Allergy declared ⇒ no substitute drug cards (SA-08).
Out of scope
Medical profile / PHR (spec 003) · Korean-language UI (Settings placeholder only) · hospital grade pin tiers (post DEV-203). Map tiles are a placeholder for the NAVER base map. astryx mapping given per token — components stay astryx-composed (HStack + NavItem tab bar per spec §1).
