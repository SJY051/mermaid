# 모바일 UI 셸

지금까지의 프론트는 "SDK가 프록시와 통한다"를 증명하는 걸어다니는 뼈대였습니다. 이 스펙은 그것을 **아픈 사람이 밤에 폰으로 쓰는 화면**으로 바꿉니다. 와이어프레임: [`wireframe-v2.html`](wireframe-v2.html) (브라우저로 열면 6개 화면).

사용자 전제가 모든 결정을 지배합니다: **아프고, 밤이고, 폰이고, 제2외국어입니다.** 인지 부하를 낮추는 쪽이 항상 이깁니다.

## 1. 정보 구조 — 하단 탭 4개

| 탭 | 내용 | 지금 있는 코드 |
|---|---|---|
| **Chat** (홈) | 메시지 리스트 · 약 카드 · 지도 프리뷰 카드 · 컴포저 | `App.tsx`의 전부가 여기로 이사 |
| **Map** | 전체 지도 · 종류/영업중 필터 · 시설 바텀시트(DEV-207) | `FacilityMap`, `NearbyFacilities` |
| **Saved** | 즐겨찾기 목록(DEV-504) · 상태 점 | `lib/storage.ts`의 `SavedFacility` |
| **Settings** | 다크 토글 · 언어(영어, 한국어는 추후) · 알레르기 프로필(opt-in) · About | `Preferences` |

- 하단 탭 바는 astryx에 없으므로 `HStack`+`NavItem`으로 직접 조립합니다(DEV-207 바텀시트와 같은 관례).
- 챗의 `OPEN_FACILITY_MAP` uiAction은 **인라인 지도 프리뷰 카드**가 되고, 탭하면 Map 탭이 그 컨텍스트(종류·반경·영업중)를 이어받습니다.

## 2. 컬러 규칙 — 색은 의미, 장식 금지

바탕은 눈이 편한 무채색(astryx neutral 테마). 유채색은 **의미가 있을 때만**, 그리고 의미당 정확히 한 색:

| 색 | 뜻 | 허용 위치 | 금지 |
|---|---|---|---|
| **빨강** | "지금 행동하라" | 응급 블록(테두리·제목·119 버튼), 알레르기 `blocked` 카드 | 그 외 전부 |
| **초록** | "지금 영업 중" | 지도 핀 채움, 상태 점 | **알레르기 관련 일체** — 초록은 "안전"으로 읽히고, `no_match_found`는 안전이 아닙니다(§2-2). 토글 on 색도 초록 금지(중립 유지) |
| **호박** | "불확실" | `Hours unknown` 핀·점, 알레르기 `warning`/`unknown` 배지 | — |
| 무채색 | 나머지 전부 | 버튼·카드·내비·로딩 | — |

`no_match_found`는 **무채색 평문**을 유지합니다 — 배지도, 색도 없이. 이건 이미 테스트가 지키는 §2-2입니다.

### 2-1. 알레르기가 선언된 턴 — 대안을 제시하지 않는다 (SA-08, 최우선)

**사용자가 알레르기를 말한 턴에는 모델이 고른 약을 화면에 절대 띄우지 않습니다.** 이건 색이 아니라 화면 구성의 문제이고, 이 앱에서 가장 미묘한 안전 함정입니다 — 나프록센 사건(이부프로펜 알레르기 사용자에게 나프록센이 추천된)이 이걸로 생겼고, `DrugContextRetriever.retrieve`가 코드로 막습니다(`withoutProposedIngredients` → 모델 제안 성분 폐기, 제품명 없으면 `allergySuppressed()` → 카드 0개).

와이어프레임이 구현 원본이므로 그림도 이 동작을 따라야 합니다:

- **약 카드는 사용자가 직접 이름을 댄 제품일 때만** 나타납니다. 모델이 "대신 이걸 드세요"로 고른 약(예: 이부프로펜 알레르기에 아세트아미노펜)은 **화면에 없습니다** — 그게 있으면 SA-08을 우회하는 것이고, 계열 교차반응 위험을 다시 여는 것입니다.
- 사용자가 특정 제품을 물었고 그게 알레르겐을 포함하면 → **`blocked` 카드(빨강)** 하나. 그 옆에 대안 카드는 없습니다.
- 증상만 말하고 제품명이 없으면 → **카드 0개.** 어시스턴트는 "알레르기를 말해줬으니 약을 직접 제안하지 않겠다"고 설명하고 **약사에게** 안내합니다(지도/약국 CTA).
- 이 "대안을 제시하지 않음"은 버그가 아니라 기능입니다 — 화면이 그 의도를 말로 밝힙니다("No medicine is suggested here — that's on purpose").

## 3. 지도 핀 — 모양=종류, 채움=상태

두 정보 축을 두 시각 축으로 분리합니다:

- **모양**: ● 약국 · ■ 병원 · ◆ 응급실
- **채움**: 초록=영업 중 · 회색=닫음 · 호박=시간 미상 (`isOpenNow: null` — §2-3, 절대 "닫음"으로 그리지 않음)
- 지도 상단에 상시 미니 범례. 같은 상태 점을 Saved 목록·바텀시트에도 재사용해 어휘를 통일합니다.
- **필터 축을 분리합니다:** 종류(All/약국/병원/응급실)는 세그먼트 컨트롤, **"Open now"는 별도 토글**입니다 — 둘은 다른 질의 파라미터(`type` vs `open_now`)이므로 같은 컨트롤에 섞지 않습니다. 세그먼트의 선택 상태는 지도에 그려진 핀과 반드시 일치합니다(예: "Pharmacies" 선택 → 약국 핀만; 모든 종류를 보이려면 "All"을 선택).
- **"Open now" 토글은 클라이언트 분할로 구현합니다 — 백엔드 `open_now=true`를 쓰지 않습니다.** 이유: 백엔드 필터는 `Boolean.TRUE.equals(isOpenNow)`라 **`isOpenNow == null`(시간 미상)을 조용히 버립니다**(`FacilityService.pharmacies`). 그걸 그대로 쓰면 "영업 중일 수도 있는" 약국이 소리 없이 사라져 §2-3을 정면으로 위반합니다. 대신 **미필터로 한 번 받아** 클라이언트에서 나눕니다:
  - `isOpenNow === true` → 지도에 **초록 핀**
  - `isOpenNow === null` → 지도에서 빼되 **버리지 않고**, "Hours unknown — call to confirm (N)" 라벨 구역(바텀시트/목록)에 둡니다
  - `isOpenNow === false`(확인된 닫힘)만 숨깁니다
- 토글이 꺼진 기본 상태에서는 세 상태 핀이 모두 보입니다(와이어프레임 프레임 4가 이 상태이고, 종류 세그먼트는 "All" — 그래서 혼합 모양·혼합 채움 핀이 정당).
- 병원 등급·분과별 세분(상급종합/의원 등)은 **DEV-203 데이터가 들어온 뒤** 후속으로 — 지금 스펙 밖.

## 4. 안전 요소의 형태 (§2 매핑)

- **면책(§2-1)**: 대형 Banner → **컴포저 위 상시 슬림 스트립**으로. "항상 화면에 보인다"는 그대로 충족하면서 모바일 화면을 돌려받습니다. 스트립은 모든 탭에 존재하고, 스크롤·상태와 무관하게 고정입니다.
  - **슬림하다고 흐릿하면 안 됩니다.** 안전 문구는 WCAG AA(본문 4.5:1)를 **넘겨야** 합니다 — 안 읽히는 면책은 없는 면책이고, 이건 Review guidelines가 P1으로 올린 "안전 정보를 가리는 접근성 결함"입니다. 와이어프레임 실측: light 5.98:1, dark 5.42:1(둘 다 11px). 회색조 `--dim`(#8a8f98, 흰 배경 3.25:1)은 안전 문구에 쓰지 마세요 — 보조 메타데이터 전용.
  - **"안전 문구"의 범위:** 면책, 응급 안내(119·최근접 응급실 방향), 로딩 안심 문구("멈춘 게 아니다" — 이탈 방지), 그리고 **영업 상태 텍스트("Open now/Hours unknown/Closed") 전부.**
  - **상태 색 점은 중복 신호이지 대체물이 아닙니다.** 초록/호박/회색 점을 텍스트 옆에 붙여도 그 텍스트는 여전히 AA를 넘겨야 합니다 — 색만으로는 색맹 사용자에게 "영업 중"과 "시간 미상"이 구별되지 않으므로, **접근 가능한 전달자는 텍스트**입니다. (이 규칙은 초안에서 "점이 옆에 있으면 메타데이터 허용"으로 잘못 적었다가 바로잡은 것입니다.)
- **응급(§2-4 결과)**: `urgency: emergency`면 채팅 안에 **빨간 전면 블록** — 제목·`tel:119` 버튼·최근접 응급실 한 줄. 화면에서 가장 큰 시각 사건이어야 합니다.
- **세션(§2-5)**: 우상단 메뉴는 "New conversation" / **"This tab's conversation"** 두 항목 + 프라이버시 부속 문구. 저장 방식이 바뀌면 이 메뉴가 목록으로 자라는 자리입니다 — 지금은 자리만.
  - **문구는 "저장하지 않는다"이지 "전송하지 않는다"가 아닙니다.** 사용자 메시지는 답을 받기 위해 `/api/v1/chat/completions`로 **전송됩니다**(그래야 LLM이 답하니까). §2-5가 막는 것은 **영속 저장**입니다 — 대화는 `sessionStorage`에만 있다가 탭과 함께 소멸하고, 서버 DB엔 저장되지 않습니다. "never uploaded / 서버로 가지 않는다"류는 전송을 부정하는 **거짓 프라이버시**이므로 금지. 정확한 표현: *"보내서 답을 받지만, 이 대화는 저장하지 않는다 — 탭을 닫으면 사라진다."*
  - **같은 함정이 알레르기 토글에도 있습니다.** "Remember my allergies" OFF는 **저장(persist)** 을 끄는 것이지 전송을 끄는 게 아닙니다 — 세션 알레르기는 `mermaid.exclude_ingredients`로 **서버에 전송되어** 그 턴의 약을 걸러냅니다(`MermaidRequestExtension`). "off면 전송 안 됨"류 금지. **범위는 "한 답변"이 아니라 "탭 전체"입니다** — `ChatSession.allergies`는 `sessionStorage`에 살아 탭 수명 동안 유지되고 그 탭의 모든 답변을 거릅니다(`storage.ts`). "이번 답에만 쓰고 잊는다"로 쓰면 후속 질문에서 SA-08 차단이 풀립니다. 정확한 표현: *"이 탭 대화 내내 쓰고, 탭을 닫으면 사라진다 — 기기에도 서버에도 저장하지 않는다."*
- **출처(§2-9) = 서명 요소**: 모든 사실 카드(약·시설)에 `✓ 기관 · 날짜` 칩. 이 제품의 논지("검증됨")를 시각 정체성으로. `dataStatus: fixture`면 상단에 `sample data` 칩.
- **100초 대기**: PR #31의 로딩·에러·재시도가 어시스턴트 버블 안으로 이사합니다. 동작 불변, 위치만.

## 5. 다크 모드

Settings 토글: 기본은 기기 설정(`prefers-color-scheme`) 추종, 토글로 고정 가능. astryx `useTheme`가 미디어쿼리를 이미 읽는 것은 확인했고, **theme-neutral의 다크 토큰 제공 범위는 스캐폴드 첫 단계에서 확인**합니다 — 없으면 이 항목만 후순위로 미룹니다(UNKNOWN을 안고 짓지 않기).

## 6. 이행 계획

1. **셸 스캐폴드**: 탭 내비 + 4 화면 골격 + 면책 스트립 (기능 이동 없이, 테스트 골격 포함)
2. **챗 이식**: `App.tsx` → Chat 탭, astryx `Chat` 킷(컴포저·메시지 리스트)으로 교체, 기존 62개 테스트 유지·수정
3. **지도 이식**: Map 탭 + 핀 모양/채움 + DEV-207 바텀시트
4. **약 카드(#16)**: 새 셸 안에서 — 컬러 규칙·출처 칩 적용
5. Saved / Settings 화면
6. **A-4**: 셸이 서면 README 히어로 스크린샷

각 단계가 PR 하나, CI 초록 + 뮤테이션 체크 + 브라우저 클릭이 완료 조건입니다(AGENTS §4).
