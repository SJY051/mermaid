## Attack Path Steps

1. Attacker or external source controls the validated input: An external model controls answer-level summary, guidance, warning, and urgency prose after receiving client and retrieved context.
2. The stated preconditions hold: A model or compatible provider emits structurally valid harmful prose that avoids the narrow lexical checks.
3. The closest control is reached but does not close the path: AnswerValidator enforces structure, citations, and markup but does not semantically bind general prose to retrieved facts or prohibit diagnoses/doses in words.
4. The input reaches the validated sink: Validated model prose is preserved by ChatProxyController and rendered by ChatScreen.
5. The resulting consequence is: A model can present a diagnosis, overdose instruction, or unverified medical claim as the application's answer.
