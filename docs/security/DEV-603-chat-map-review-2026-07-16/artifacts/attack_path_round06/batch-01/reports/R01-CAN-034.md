## Attack Path Steps

1. Attacker or external source controls the validated input: An external model controls the top-level answerId string.
2. The stated preconditions hold: The model emits the reserved answerId in an otherwise valid response and the user interacts with the picker.
3. The closest control is reached but does not close the path: The provider schema and validator do not reserve the server-only allergy clarification identifier.
4. The input reaches the validated sink: ChatScreen interprets the magic answerId as an instruction to open the allergen picker; confirmation rewrites authoritative allergy session state.
5. The resulting consequence is: A model can invoke a server-only safety workflow and cause the user to replace or stamp allergy state without the server detecting a declaration.
