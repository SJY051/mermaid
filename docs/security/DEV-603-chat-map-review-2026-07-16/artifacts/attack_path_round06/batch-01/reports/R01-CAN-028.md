## Attack Path Steps

1. The affected legitimate user supplies reaction-only allergy wording; no distinct attacker controlling another user’s request is established.
2. The stated preconditions hold: The declaration uses reaction-only wording outside the recognized keyword set and is included in the screened user text.
3. The closest control is reached but does not close the path: AllergyDeclaration recognizes only explicit allergy, anaphylaxis, or intolerance keywords.
4. The input reaches the validated sink: The declaration is missed, so DrugContextRetriever performs model ingredient extraction and medicine retrieval.
5. The resulting consequence is: Model-proposed medicines can be shown despite a declared reaction, bypassing SA-08's fail-closed allergy gate.
