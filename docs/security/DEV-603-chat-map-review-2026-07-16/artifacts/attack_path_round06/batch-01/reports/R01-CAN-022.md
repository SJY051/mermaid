## Attack Path Steps

1. Attacker or external source controls the validated input: An anonymous caller controls the number and size of chat messages and their content.
2. The stated preconditions hold: The public endpoint is reachable and upstream/provider limits do not reject the body before meaningful local or billable work.
3. The closest control is reached but does not close the path: The controller accepts raw JsonNode; service code joins and forwards all allowed message content with no request-size or history budget.
4. The input reaches the validated sink: Blocking extraction and answer calls receive the unbounded input under server credentials.
5. The resulting consequence is: One request can consume excessive memory, model tokens, provider quota, and request-worker time, reducing care-information availability.
