## Attack Path Steps

1. Attacker or external source controls the validated input: An external model controls emergency urgency title and message for answers that were not short-circuited by deterministic triage.
2. The stated preconditions hold: The model emits emergency level and valid action structure but misleading title/message text.
3. The closest control is reached but does not close the path: The controller preserves model urgency copy and the validator checks action presence and markup, not safety meaning.
4. The input reaches the validated sink: ChatScreen and MobileShell render the model text as the authoritative red emergency alert.
5. The resulting consequence is: Contradictory or minimizing model copy can tell a user to wait despite an emergency classification.
