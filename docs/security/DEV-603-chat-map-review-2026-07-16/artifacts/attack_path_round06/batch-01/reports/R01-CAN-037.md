## Attack Path Steps

1. Attacker or external source controls the validated input: An external or slow facility API controls response latency after a model map action or direct lookup.
2. The stated preconditions hold: A facility request is pending long enough for the initial render to be visible.
3. The closest control is reached but does not close the path: NearbyFacilities initializes facilities as an empty list and has no loading state while the request is pending.
4. The input reaches the validated sink: The render branch displays the definitive no-results message before the request completes.
5. The resulting consequence is: A sick user can be told no places were found when the application has not finished looking and may abandon care navigation.
