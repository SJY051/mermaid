## Attack Path Steps

1. The affected legitimate user supplies an emergency description from an omitted category; no distinct attacker controlling another user’s request is established.
2. The stated preconditions hold: The user's wording describes an omitted emergency category and does not coincidentally match an existing rule.
3. The closest control is reached but does not close the path: EmergencyTriage's own TODO acknowledges missing categories and its fixed regex list omits them.
4. The input reaches the validated sink: The request falls through to allergy clarification or ordinary retrieval and LLM handling.
5. The resulting consequence is: A time-critical emergency can receive routine medicine or clarification UI instead of immediate server-authored emergency action.
