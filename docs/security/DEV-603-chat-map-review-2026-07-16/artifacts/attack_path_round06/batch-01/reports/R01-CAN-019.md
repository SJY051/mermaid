## Attack Path Steps

1. Attacker or external source controls the validated input: An external model authors drug-card indication, caution, interaction, and similar English translations.
2. The stated preconditions hold: The model emits plausible structurally valid but semantically false translations for a retrieved product.
3. The closest control is reached but does not close the path: The controller checks nonblank and digit/shape properties, while AnswerValidator binds card identity and citations but not translation meaning.
4. The input reaches the validated sink: Model-authored translations are returned in provenance-bearing DrugCard components.
5. The resulting consequence is: False or inverted medical guidance can appear inside an official-looking verified drug card.
