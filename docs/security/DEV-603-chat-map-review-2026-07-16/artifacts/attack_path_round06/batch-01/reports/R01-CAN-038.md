## Attack Path Steps

1. Attacker or external source controls the validated input: A successful server allergen-options response controls the currently available canonical key set after a user already has saved or session allergy keys.
2. The stated preconditions hold: A previously selected key is absent from a later successful options response and the user confirms the picker.
3. The closest control is reached but does not close the path: AllergenPicker filters existing selectedKeys to the refreshed availableKeys and later serializes only current options.
4. The input reaches the validated sink: On confirmation, chatSession replaces authoritative allergy arrays with the reduced list.
5. The resulting consequence is: A previously declared allergen can disappear silently, after which medicine retrieval may proceed on an incomplete exclusion set and show no_match_found.
