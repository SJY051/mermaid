## Attack Path Steps

1. Attacker or external source controls the validated input: Prompt- or repository-influenced command text embeds source .env inside a child shell.
2. The stated preconditions hold: A populated readable .env exists, the hook is active, and the nested shell runs.
3. The closest control is reached but does not close the path: The shallow segmenter checks source only at the outer token level and does not parse shell -c programs.

5. The resulting consequence is: Credentials become available to attacker-selected follow-on commands and can be exfiltrated.
