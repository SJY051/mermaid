## Attack Path Steps

1. Attacker or external source controls the validated input: Government API narrative fields are external, untrusted strings controlled by the upstream record or any compromised intermediary/cache.
2. The stated preconditions hold: A government record, fixture, cache, or upstream path contains instruction-like narrative and the model follows it.
3. The closest control is reached but does not close the path: The retriever serializes those strings into context and ChatProxyService elevates the whole context as a system-role message.
4. The input reaches the validated sink: The model consumes upstream narrative with system privilege and its semantically unchecked prose can reach the response.
5. The resulting consequence is: Prompt-like government content can steer the model to emit unverified or unsafe medical text under the application's authority.

## Attack Path Facts

- Context: Not self-only if an affected external record is reused: government/public-data text crosses into a system-role model message and then the verified-answer UI, potentially influencing every requester who retrieves that record.
