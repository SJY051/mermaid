## Attack Path Steps

1. Source: Unauthenticated GET /api/v1/geocode with a nonblank address up to 200 characters.
2. Preconditions: The geocode route is internet reachable in production. Naver meters, charges, or throttles the configured credential at a practically relevant level.
3. Control gap: Input length and a five-result output cap exist; no endpoint rate, cache, or request coalescing control exists.
4. Sink: Every accepted query sends a new credentialed Naver geocoding request.
5. Impact: Automated unique queries can consume the application's provider quota or billable capacity and degrade map search.
