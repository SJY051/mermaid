## Attack Path Steps

1. Source: Two callers in the same 0.001-degree cache cell with different exact origins.
2. Preconditions: Two searches share a rounded key before the six-hour TTL expires. Provider distances are origin-dependent and at least one result lies near an ordering/radius boundary.
3. Control gap: Rounding bounds cache cardinality, but the cached batch retains provider distance values computed for the first caller and carries no canonical origin correction.
4. Sink: Later radius filtering, ordering, and displayed distance trust the first caller's distances.
5. Impact: A later user can see wrong nearest ordering or have a pharmacy incorrectly included/excluded near a radius boundary.
