## Attack Path Steps

1. Source: Anonymous detail/chat request for an attacker-selected itemSeq during HYBRID fallback.
2. Preconditions: HYBRID permission lookup falls back for a requested itemSeq different from the fixture row. The composed result reaches a card or grounding path.
3. Control gap: The permission client takes the first fixture row without checking ITEM_SEQ equality.
4. Sink: DrugService uses the requested ID/source while copying fixture identity, ingredients, status, and allergy-relevant fields.
5. Impact: Wrong product facts and allergy inputs are server-authored for another medicine, violating §2-9 and potentially undermining §2-2 safeguards.
