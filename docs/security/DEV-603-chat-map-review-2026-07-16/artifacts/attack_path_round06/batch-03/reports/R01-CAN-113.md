## Attack Path Steps

1. Source: Drug detail/chat request during HYBRID DUR failure for an arbitrary itemSeq.
2. Preconditions: A DUR operation fails in HYBRID for an itemSeq not represented by its fixed fixture. The requested drug detail is still composed and returned/grounded.
3. Control gap: Each warning kind loads a fixed fixture and the parser never requires returned ITEM_SEQ to equal the requested medicine.
4. Sink: Fixed combination/age/elderly rows or an empty pregnancy fixture are attached as the requested product's complete warnings.
5. Impact: The server can invent irrelevant contraindications or silently omit real ones for another medicine, routing around verified safety/provenance controls.
