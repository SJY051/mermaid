## Attack Path Steps

1. Source: Anonymous GET /api/v1/drugs/{itemSeq} with an unconstrained caller-selected identifier.
2. Preconditions: The detail route is exposed without an external admission control. Live/hybrid clients are configured and attacker identifiers miss relevant caches.
3. Control gap: Parallel assembly reduces latency and caches completed results, but itemSeq has no format/rate admission and empty permission/narrative results are not cached.
4. Sink: One cold identifier starts permission, EasyDrug, and four DUR operations before existence is known.
5. Impact: Distinct or repeated nonexistent identifiers can multiply provider usage and occupy request threads, degrading drug and chat availability.
