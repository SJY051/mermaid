## Affected Locations

1. Path: backend/src/main/java/com/mermaid/drug/DrugController.java; lines: 36-50; label: entrypoint; detail: The anonymous search route accepts raw exclude_ingredients and forwards only normalizeAvoided output.
2. Path: backend/src/main/java/com/mermaid/drug/DrugController.java; lines: 61-76; label: root_control; detail: A null normalization key is omitted without an error, warning, or unverified-allergen representation.
3. Path: backend/src/main/java/com/mermaid/drug/AllergyChecker.java; lines: 39-43; label: broken_safety_default; detail: If every exclusion was omitted, the checker returns no_match_found because it sees an empty avoided set.
4. Path: backend/src/main/java/com/mermaid/drug/DrugService.java; lines: 98-124,339-352; label: sink; detail: Search summaries are returned with allergy verdicts computed from the silently reduced set.

## Attack Path Steps

1. An unauthenticated client calls GET /api/v1/drugs with a syntactically accepted exclude_ingredients value that cannot be normalized.
2. DrugController silently omits the unbound exclusion instead of rejecting or preserving uncertainty.
3. DrugService evaluates returned products against the reduced avoided set.
4. If all exclusions were dropped, AllergyChecker returns no_match_found and the server releases that verdict with the official product response.
