## Affected Locations

1. Path: frontend/src/lib/storage.ts; lines: 46-84,108-129; label: source; detail: The page origin holds a persistent deviceId, consultation messages/session allergies in sessionStorage, and opt-in allergy preferences in localStorage.
2. Path: frontend/src/hooks/useNaverMap.ts; lines: 55-85; label: root_control; detail: Opening a map appends a third-party script element directly to document.head; it is not isolated in a sandboxed/cross-origin iframe.
3. Path: frontend/src/hooks/useNaverMap.ts; lines: 121-135; label: sink; detail: The fetched SDK runs in the application document and receives normal page JavaScript authority before the map is marked ready.
4. Path: frontend/src/components/MobileShell.tsx; lines: 29-46; label: concrete_implementation; detail: Chat and Map remain mounted in the same top-level document and the SDK loads when the Map tab becomes active.

## Attack Path Steps

1. A user opens Map, causing the application to append the fixed Naver SDK URL to document.head.
2. The remote SDK executes in the top-level application document with normal first-party JavaScript authority.
3. If the vendor or delivery path is compromised, that code can read deviceId, consultation/session-allergy state, and opt-in preferences and can issue same-origin API calls or alter the UI.

### Mitigations Already Present

- The SDK URL is fixed HTTPS.
- Loading is gated on opening Map.
- Chat uses sessionStorage and allergy persistence is opt-in.
- Authentication failure is registered before script append.

### Secrets References

- The browser deviceId is a bearer-like profile capability; consultation and opt-in allergy state are sensitive data rather than server credentials.

### Blindspots

- Production CSP and browser isolation policy are not established by repository evidence.
- Whether a separate-origin iframe is compatible with product requirements requires human architecture review.
- The probability of vendor or delivery-chain compromise is not established.

### Controls

- Fixed HTTPS vendor URL.
- Map-on-demand loading.
- Session-scoped chat storage.
- Opt-in allergy persistence.

## Counterevidence
