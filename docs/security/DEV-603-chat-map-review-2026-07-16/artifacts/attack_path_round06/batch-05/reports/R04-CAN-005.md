# R04-CAN-005 — Positive itemSeq detail caches have attacker-expandable cardinality

- Candidate ID: R04-CAN-005

- Instance key: drug-detail-cache:positive-itemseq-cardinality
- Ledger row ID: central-validation-round-06:R04-CAN-005

## Attack Path Steps

1. Source: An anonymous caller chooses arbitrary itemSeq values on the public drug-detail route.
2. Control gap: Positive permissionDetail and easyDrugBySeq values live six hours with no route admission, cache cardinality, Redis maxmemory, or eviction budget.
3. Sink: Distinct itemSeq values create Redis entries in both positive detail caches; HYBRID fallback can make the positive keyspace query-independent.
4. Impact: Enumeration or fallback-driven misses consume Redis memory, evict useful state, and degrade availability.
