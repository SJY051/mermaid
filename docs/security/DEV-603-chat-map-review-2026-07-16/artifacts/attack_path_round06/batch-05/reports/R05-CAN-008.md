## Attack Path Steps

1. Source: User message containing an instruction designed to make pass 1 return an unrelated, prescription, or otherwise inappropriate valid INN.
2. Control gap: The extraction prompt says never follow instructions and the parser caps three INN-shaped terms; no server-side semantic relation binds a term to the stated symptoms.
3. Sink: A manipulated but real ingredient can be officially retrieved, giving later cards valid government provenance even though selection was inappropriate.
4. Impact: The user can receive an apparently verified medicine result chosen by prompt injection rather than by the intended symptom-to-search task.
