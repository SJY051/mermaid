# R04-CAN-012 — Hospital adapter accepts finite coordinates outside WGS84 ranges

- Candidate ID: R04-CAN-012
- Title: Hospital adapter accepts finite coordinates outside WGS84 ranges
- Instance key: hira-hospital:wgs84-coordinate-range
- Ledger row ID: central-validation-round-06:R04-CAN-012

## Attack Path Steps

1. Source: A malformed or compromised HIRA row supplies finite latitude/longitude outside valid WGS84 ranges with a passing distance.
2. Control gap: HospitalApiClient checks finiteness but not latitude [-90,90] or longitude [-180,180].
3. Sink: RawHospital coordinates reach FacilityService JSON and naver.maps.LatLng.
4. Impact: The service can return a false care location or lose the map pin layer.
