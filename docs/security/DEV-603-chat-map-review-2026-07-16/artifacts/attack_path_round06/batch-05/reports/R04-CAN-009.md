## Attack Path Steps

1. Source: Browser geolocation returns exact device coordinates.
2. Control gap: The user grants browser permission and the SDK host is fixed HTTPS, but the UI does not identify Naver as a recipient.
3. Sink: The coordinates are passed to Naver-controlled Map/LatLng/setCenter code.
4. Impact: Third-party code processes precise location and may associate map activity with browser or network identifiers.
