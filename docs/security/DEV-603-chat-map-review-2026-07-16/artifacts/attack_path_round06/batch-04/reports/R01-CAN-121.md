## Attack Path Steps

1. Actor/source: Unauthenticated GET requests with attacker-selected distinct deviceId values.
2. Control failure: ProfileService.read creates a durable row on every miss with no creation budget, rate control, or expiry.
3. Boundary/sink: MariaDB user_profile and its unique index receive one row per distinct identifier.
4. Consequence: Sustained requests can exhaust storage/write/index capacity and degrade shared profile availability.
