## Attack Path Steps

1. Public ingress: backend/src/main/java/com/mermaid/profile/ProfileController.java:24-27 — The public bearer-deviceId GET route returns the migrated profile through ProfileService.read.
2. Actor/source: Allergy rows created under V1 before consent existed.
3. Control failure: V2 sets existing profiles to consent false without deleting, quarantining, or reconciling existing child rows.
4. Boundary/sink: backend/src/main/java/com/mermaid/profile/dto/ProfileDtos.java:18-33 — ProfileResponse serializes legacy allergy rows even while rememberAllergies is false.
5. Consequence: After migration, legacy health data remains stored and disclosed while memory is reported disabled.
