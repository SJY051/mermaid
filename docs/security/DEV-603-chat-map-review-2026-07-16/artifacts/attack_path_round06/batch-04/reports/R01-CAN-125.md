## Attack Path Steps

1. Public ingress: backend/src/main/java/com/mermaid/profile/ProfileController.java:42-53 — The public bearer-deviceId API exposes consent update and allergy creation as independently callable HTTP requests.
2. Actor/source: Concurrent opt-out and allergy-create requests for one initially opted-in deviceId.
3. Control failure: Separate stale transactions lack row locks, optimistic versioning, and a database consent/child constraint.
4. Boundary/sink: backend/src/main/java/com/mermaid/profile/dto/ProfileDtos.java:18-33 — Subsequent profile reads serialize retained allergy rows without a consent gate.
5. Consequence: Health data may remain stored and disclosed after consent withdrawal.
