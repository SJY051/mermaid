## Attack Path Steps

1. Actor/source: Unauthenticated consent and allergy-create requests against a selected profile.
2. Control failure: No per-profile allergy count exists; each distinct value is appended and the full collection remapped.
3. Boundary/sink: MariaDB allergy rows grow and ProfileResponse serializes the complete list.
4. Consequence: Repeated additions can consume storage and create growing read/mutation work, degrading profile availability.
