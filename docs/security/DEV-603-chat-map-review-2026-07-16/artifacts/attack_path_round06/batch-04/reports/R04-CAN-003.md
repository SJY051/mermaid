## Attack Path Steps

1. Actor/source: Dependency installation in a detected Git checkout where local hooksPath configuration fails.
2. Control failure: Installer converts in-checkout git-config failure to warnings and successful exit.
3. Boundary/sink: Later commits run without .githooks/pre-commit when the warning is missed.
4. Consequence: The secret guard can be absent while setup appears successful, increasing risk of credential commits and later publication.
