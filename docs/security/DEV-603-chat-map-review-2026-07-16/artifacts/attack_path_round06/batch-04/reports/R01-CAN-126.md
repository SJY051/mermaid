## Attack Path Steps

1. Public ingress: backend/src/main/java/com/mermaid/profile/ProfileController.java:30-45 — The public bearer-deviceId API exposes country update and consent update as independently callable HTTP requests.
2. Actor/source: Concurrent country-update and consent opt-out requests after both load consent enabled.
3. Control failure: The same non-versioned entity has no locking or @DynamicUpdate, so default ORM SQL may write a stale consent column.
4. Boundary/sink: backend/src/main/resources/db/migration/V2__profile_optin_and_normalisation.sql:21-24 — The database consent bit can be restored by the later stale parent update.
5. Consequence: Withdrawn health-data consent may be silently restored, enabling later retention.
