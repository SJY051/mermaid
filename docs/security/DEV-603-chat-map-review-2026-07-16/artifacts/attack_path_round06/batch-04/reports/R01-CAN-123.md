## Attack Path Steps

1. Actor/source: Unauthenticated favorite-create requests using distinct attacker-selected facility IDs.
2. Control failure: No per-profile favorite count or known-facility requirement exists.
3. Boundary/sink: MariaDB favorite rows grow and profile reads serialize every row.
4. Consequence: Repeated additions can exhaust storage and make profile operations increasingly expensive.
